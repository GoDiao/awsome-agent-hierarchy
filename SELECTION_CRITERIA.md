# Selection Criteria

Awesome AI Agent Hierarchy is organized as an architectural map. The goal is to help readers understand the agent stack quickly and find representative repositories for each capability.

## Curation Standard

Each third-level topic may list **up to 10 curated GitHub repositories**. The list does not need to be filled to 10. Quality, relevance, and maintainability are more important than symmetry.

## Inclusion Signals

- Direct relevance to the topic and to AI agent systems.
- Clear source code, documentation, or reproducible examples.
- Active maintenance, ecosystem adoption, or official ownership.
- Unique technical value that is not already represented by a stronger project in the same topic.
- Research or benchmark value for important agent techniques.

## Exclusion Signals

- Broken, private, or inaccessible repositories.
- Thin wrappers with little original value.
- Duplicates within the same topic.
- Projects unrelated to agent systems, even if they are broadly AI-related.
- Archived projects without historical or educational value.

## Tagging Standard

Every repository item must include one or more of:

`Official`, `Framework`, `Example`, `Research`, `Infra`, `Archived Classic`

Use `Archived Classic` only when the repository is archived or inactive but intentionally retained.
